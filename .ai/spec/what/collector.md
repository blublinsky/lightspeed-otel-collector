# OTel Collector

Custom OpenTelemetry Collector for OpenShift Lightspeed. Built with the OpenTelemetry Collector Builder (OCB). See parent spec `ols/.ai/spec/what/templog.md` for cross-repo requirements and architecture.

## Behavioral Rules

### Build

1. The Collector binary is built with OCB using a build manifest (`builder-config.yaml`). The manifest declares exactly which components are compiled into the binary.
2. The build manifest includes:
   - **Receiver:** `otlpreceiver` (standard OTLP gRPC receiver from `go.opentelemetry.io/collector/receiver/otlpreceiver`)
   - **Exporters:**
     - `postgresexporter` (custom, this repo) — writes OTLP log records to PostgreSQL
     - `otlpexporter` (standard, from `go.opentelemetry.io/collector/exporter/otlpexporter`) — forwards OTLP traces to an external endpoint
3. No processors are included. The Collector receives and exports — no transformation.
4. The resulting binary is a single statically-linked Go executable.

### Configuration

5. The Collector does not ship a default configuration. Its configuration is generated by the lightspeed-operator and mounted as a ConfigMap at runtime.
6. The Collector reads its configuration from a YAML file path specified via the `--config` flag.
7. The configuration defines up to two pipelines:
   - **Logs pipeline:** `otlp` receiver → `postgres` exporter. Active when `OLSConfig.spec.templog` is `true`.
   - **Traces pipeline:** `otlp` receiver → `otlp/traces` exporter. Active when `OLSConfig.spec.audit.otel.endpoint` is set.
8. When neither pipeline is active, the Collector runs with an empty pipeline configuration. It accepts connections but does not export data.

### Deployment

9. The Collector runs as a single-replica Deployment managed by the lightspeed-operator.
10. The Collector listens on port 4317 for OTLP gRPC connections.
11. The Collector connects to PostgreSQL using credentials from a shared secret managed by the lightspeed-operator. The DSN is injected via environment variable (`POSTGRES_DSN`).
12. TLS between the Collector and PostgreSQL uses service-ca certificates (same as all other Postgres clients in the system).
13. A NetworkPolicy restricts ingress to port 4317 from agentic-operator and sandbox pods only.

### Health

14. The Collector exposes the standard OTel Collector health check extension on port 13133 (`/`). The lightspeed-operator uses this for liveness and readiness probes.

## Container Image

15. The Dockerfile builds the OCB binary in a multi-stage build:
    - **Stage 1:** Go build stage — runs `ocb` with the build manifest to produce the Collector binary.
    - **Stage 2:** Runtime stage — minimal base image (UBI 9 micro or distroless) with the Collector binary.
16. The image is built and shipped via Konflux.
17. The image tag is managed by the lightspeed-operator (not user-configurable).

## Configuration Examples

### Full configuration (templog + trace forwarding)

```yaml
receivers:
  otlp:
    protocols:
      grpc:
        endpoint: 0.0.0.0:4317

exporters:
  postgres:
    dsn: "${POSTGRES_DSN}"
    schema: templogs
    table: logs
  otlp/traces:
    endpoint: "${TRACE_ENDPOINT}"
    tls:
      insecure: false

extensions:
  health_check:
    endpoint: 0.0.0.0:13133

service:
  extensions: [health_check]
  pipelines:
    logs:
      receivers: [otlp]
      exporters: [postgres]
    traces:
      receivers: [otlp]
      exporters: [otlp/traces]
```

### Templog only (no trace forwarding)

```yaml
receivers:
  otlp:
    protocols:
      grpc:
        endpoint: 0.0.0.0:4317

exporters:
  postgres:
    dsn: "${POSTGRES_DSN}"
    schema: templogs
    table: logs

extensions:
  health_check:
    endpoint: 0.0.0.0:13133

service:
  extensions: [health_check]
  pipelines:
    logs:
      receivers: [otlp]
      exporters: [postgres]
```

### Neither active (Collector running, no export)

```yaml
receivers:
  otlp:
    protocols:
      grpc:
        endpoint: 0.0.0.0:4317

extensions:
  health_check:
    endpoint: 0.0.0.0:13133

service:
  extensions: [health_check]
  pipelines:
    logs:
      receivers: [otlp]
      exporters: []
```
